# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

Thai-French voice + text translator. The browser captures audio or text, sends it to a local FastAPI server, which runs STT → LLM translation → TTS and returns the translated audio.

## Commands

### Backend
```bash
cd backend
source .venv/bin/activate
python main.py          # starts on https://0.0.0.0:8000
```

### Frontend
```bash
cd frontend
npm run dev             # starts on https://0.0.0.0:5173
npm run build           # production build → dist/ (includes PWA service worker)
```

### First-time setup (installs deps + generates TLS cert)
```bash
bash setup.sh
```

## Architecture

```
Browser (React/Vite :5173)
  ├─ Text input  →  POST /translate  →  { translation }
  └─ Mic button  →  POST /voice (audio/webm)
                          ↓
                  FastAPI (:8000)
                  ├─ stt.py   : faster-whisper  (audio → text + language code)
                  ├─ llm.py   : Ollama /api/chat (text → translation)
                  └─ tts.py   : edge-tts         (text → MP3 bytes)
                          ↓
Browser  ←  MP3 audio response + X-Transcript / X-Translation headers
```

## Configuration

All tuneable values live in `backend/config.py`:
- `OLLAMA_MODEL` — default `bjoernb/gemma4-e4b-fast:latest`
- `WHISPER_MODEL_SIZE` — `tiny | base | small | medium | large-v3`
- `WHISPER_DEVICE` — `cpu | cuda`
- `TTS_VOICE_THAI / TTS_VOICE_FRENCH` — edge-tts voice names
- `HOST / PORT` — server binding

The frontend reads `VITE_API_URL` from `frontend/.env` (default `https://192.168.81.126:8000`).

## HTTPS

HTTPS is required because mobile browsers block `getUserMedia()` on plain HTTP.  
Certs live in `certs/` (generated by `setup.sh` via openssl, gitignored).  
On first use from the phone, visit `https://192.168.81.126:8000` and accept the self-signed certificate.

## PWA

The frontend is a Progressive Web App installable on Android and iOS.

- **Manifest & service worker**: generated by `vite-plugin-pwa` (Workbox). Static assets are precached; API routes (`/translate`, `/voice`, `/tts`) are `NetworkOnly`.
- **Icons**: `frontend/public/` contains `icon.svg`, `pwa-192x192.png`, `pwa-512x512.png`, `apple-touch-icon.png`.
- **Build note**: `npm run build` passes `NODE_OPTIONS=--experimental-global-webcrypto` for Node 18 compatibility with Workbox's minifier.
- **iOS install**: Share → "Sur l'écran d'accueil". Mic permission is NOT persistent on iOS across sessions (browser limitation).
- **Android install**: Chrome shows an "Ajouter à l'écran d'accueil" banner automatically.

## Key design notes

- **Language detection**: faster-whisper returns the detected language code (`th`/`fr`) for voice input. For text input, the LLM prompt instructs the model to detect and translate automatically.
- **Non-ASCII headers**: `X-Transcript` and `X-Translation` response headers are URL-encoded (`urllib.parse.quote`) because HTTP headers must be ASCII; the frontend decodes them with `decodeURIComponent`.
- **Model lazy-loading**: the Whisper model is loaded once on first request and cached in `stt._model` to avoid reloading on each call.
- **Mic stream lifecycle**: `TranslatorInput` requests `getUserMedia` once on mount and keeps the `MediaStream` alive in `streamRef` across recordings. This avoids re-asking for mic permission during a session. A `pendingStopRef` flag handles the edge case where the user releases the mic button while a permission dialog is still open.
